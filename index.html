Harvey: A Second Face for Your JavaScript

Harvey executes certain parts of your JavaScript depending on the current deviceâ€™s type, screen size, resolution, orientation, or any other of the same media query conditions you can use in CSS.

Responsive JavaScript—finally

Responsive and mobile first web design is inarguably a great way to build futureproof websites and applications. But the one thing CSS media queries can’t always help us with is improving the way users interact with our interfaces. Harvey helps you adjusting your JavaScript to optimize the user’s experience for their output device whenever necessary.

Small footprint
The script weighs only 1k gzipped and has no external dependencies. Also it is originally written in CoffeeScript.
…As easy as flipping a coin
Harvey has a straight-forward public API with just two static methods that still allow for unlimited customization.
Broad browser support
Polyfills and browser fixes are automatically applied wherever necessary so you don’t need to worry about them.

What does Harvey look like?

The only two methods you need to completely control Harvey are Harvey.attach() and Harvey.detach(). When attaching a condition to Harvey (in the form of a valid CSS media query), you can also register up to three callbacks for it:

Harvey.attach( 'screen and (min-width: 40em)', { setup: function() {}, on: function() {}, off: function() {} }); You can repeat this step multiple times and for any valid media query you need.

When to use callbacks

setup
Called the first time the media query becomes valid. Tip: Use the setup callback to load extra assets or manipulate the DOM.
on
Called every time the media query becomes valid. Tip: Use the on callback to reveal changes you made (e.g. in setup).
off
Called every time the media query becomes invalid. Tip: Use the off callback to revert/hide changes you made (e.g. in setup).
